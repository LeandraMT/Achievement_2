Documentation for leMoovie Application

Purpose

The application will be called 'leMoovie' and it will provide users with access to information about all types of movies, directors, actors and genres.

Some of the main features of the application will be for the users to receive information on movies, directors, actors and genres.
This allows the users to stay up to date to new launches or learn more about the movies.
The users will also be able to create and design their profile so they can save their data about their favourite movies.

  1. Home page with all the movies available
  2. Obtain data such as description of the movie, genre, image URL etc.
  3. Allow new users to register
  4. Allow users to update their info such as; username, password, email, date of birth etc.
  5. Allow existing users to deregister
  6. Allow users to add and remove movies
Business Logic URL HTTP Method Require body data format Response body data format
Allows the users to login to homepage /login POST none Auth when logging in
Provides a list of all the movies /movies GET none JSON Object returning ALL the movies to the Homepage
Provides data about a single movie /movies/:title GET none JSON Object with description of the movie
Provides data about the genre of each movie /movies/genre/:Genre GET none JSON Object with data about the genre and its description
Provides data of the director /movies/directors/:Director GET none JSON Object with information of the director of each movie
Allow new users to register /users POST none JSON Object with welcome text and the Homepage
Allows users to make changes/updates to their profiles /users/:username PUT none JSON Object with data on the changes/updates made
Allow users to add movie to a list of favourites /users/:Username/movies/:MovieID POST none JSON Object providing data with the newly added movie
Allows users to remove a movie from their list of favourites /users/:Username/movies/:MovieID DELETE none Text message confirming the movie has been removed
Allows users to deregister /users/:username DELETE none Message confirming the deregistration to the user
